## Overview

Aether is a production-grade, multi-tenant enterprise SaaS platform structured as a monorepo using `pnpm` workspaces and managed with `turbo`. It consists of:
- **Next.js frontend client** (`apps/web`): A Next.js 16 (Turbopack) frontend utilizing React 19 and Tailwind CSS v4.
- **Express.js backend server** (`apps/api`): An Express.js 5 API server configured with security/rate-limiting/cors, and structured context tracing using `AsyncLocalStorage`.
- **Database package** (`packages/database`): Shared package defining the database schema (Prisma + PostgreSQL) and seeding scripts.
- **Shared types package** (`packages/shared-types`): Shared type interfaces and endpoints.
- **Shared config package** (`packages/config`): TSConfig, ESLint, and linting rule configurations.

## Data Flow

1. Client (`apps/web`) initiates an HTTP request containing `X-Tenant-ID` or headers/cookies.
2. Request passes through security/middleware (`helmet`, `cors`, `compression`, `cookie-parser`).
3. Request enters `contextMiddleware` inside Express (`apps/api/src/presentation/http/middleware/context.middleware.ts`), generating a new request/correlation scope with Node.js `AsyncLocalStorage`.
4. Business logic layers process request scoped to the tenant.
5. Queries execute through Prisma database client using tenant ID to isolate rows.
